워드 프레스

홈페이지

사실상 아무 사이트나 다 만들 수 있는 플랫폼. 자유도가 무한대인 대신에 진입장벽이 꽤 있음. 프로그래밍을 못하면 자유도가 조금 떨어짐 (그래도 여기 옵션 중에선 제일 높음). 사용자가 워낙 많아서 사용법 공부하기가 쉬움. 대충 각 잡고 일주일 정도는 공부해야 사용 가능할 듯??

백엔드를 직접 구현해야하는게 아주 큰 단점: 선택지가 2개가 있음.

  1. 백엔드를 따로 구현 안하고 github page 사용 (내 블로그가 하는 방식)
    • 이럴 거면 굳이 어렵게 워드 프레스 쓸 필요 없음. zola 쓰거나 차라리 내 엔진 쓰는게 나음
  2. 백엔드 공부해서 직접 호스팅
    • 백엔드 공부는 진입 장벽이 좀 있음...
  3. 호스팅 업체 사용
    • 돈이 듦 (많이는 안 들고 한달에 몇천원 정도)
    • 열심히 찾아보면 공짜도 있긴 할텐데 보통 공짜 호스팅 사이트는 제약이 많음..ㅠ
기능구현도
글 작성 방식자체 에디터 있다는데 확인 안 해봄: 네이버랑 비슷한데 자유도 조금 더 높고 난이도도 조금 더 높은 듯?
글에 넣을 수 있는 컨텐츠생각할 수 있는 모든 거 다 가능
내부 검색플러그인 사용하면 가능.
커뮤니티플러그인 사용하면 가능.
이쁘게 꾸미기생각할 수 있는 모든 거 다 가능
접근성검색 잘 되도록 하는 플러그인 찾아보면 있을 걸?
데이터 내보내기내보내기 도와주는 플러그인 찾아보면 있을 걸? 근데 블로그 글들을 텍스트 파일로 내보내고 싶으면 zola나 내 블로그 같은 SSG 사용하는게 나음
기타 단점돈이 들 확률이 높음

Zola

홈페이지

내가 옛날에 말했던 프로그램! 이거는 내가 도우미 프로그램 짜서 자기 줄 수 있어! 사실 이거 말고도 SSG (static site generator) 많은데 그냥 이걸로 했어. 어차피 SSG끼리 기능 차이는 별로 안 나고 나한테 얘가 제일 익숙하거든.

기능구현도
글 작성 방식마크다운 파일을 직접 작성 (아마 비쥬얼스튜디오코드로)
글에 넣을 수 있는 컨텐츠마크다운만 가능 (사진 넣기, 강조하기, 표 넣기 등등)
내부 검색지원함.
커뮤니티댓글/조회수/이웃 등등 -> 사실상 불가능... 백엔드 할줄 알고 모르고를 떠나서 SSG로는 저런 거 사실상 힘듦
이쁘게 꾸미기기본으로 제공해주는 테마가 꽤 있음. 근데 네이버나 워드프레스에 비해서는 아주 빈약. CSS 공부를 조금 하면 꾸미는 자유도는 아주 높아짐
접근성검색 잘 되도록하는 플러그인이 있는 거 같기는 한데 좀 빈약한 듯... 블로그 아주 활발히 운영하면 구글에 적당히 검색되는 정도?
데이터 내보내기아주 편함. 어차피 글을 마크다운으로 쓰기 때문에 그대로 갖고 가면 됨

내가 쓰는 방식

기능구현도
글 작성 방식마크다운 파일을 직접 작성 (아마 비쥬얼스튜디오코드로)
글에 넣을 수 있는 컨텐츠네이버에서 되는 건 거의 다 됨... 글꼴 빼고!
네이버에서 안 되는 것도 많이 됨: 각주[0]각주 달거나 tooltiptooltip 다는 것처럼
내부 검색백엔드가 없어서 검색/댓글/로그인/조회수확인 등등이 아무것도 안됨
커뮤니티백엔드가 없어서 검색/댓글/로그인/조회수확인 등등이 아무것도 안됨
이쁘게 꾸미기CSS 파일을 직접 수정하거나 엔진을 직접 수정해야함. 자유도는 제일 높지만 (생각할 수 있는 모든 디자인이 다 가능), 나 말고는 아무도 수정 못함
접근성아무 검색엔진에서도 안 보임... ㅎㅎ 이건 내가 아직 추가를 안해서 ㅠㅠ
지금은 내가 직접 링크를 전달해줘야 접속 가능
데이터 내보내기모든 파일이 마크다운으로 돼 있어서 데이터 관리는 아주 쉬움. 마음만 먹으면 파일들 통째로 갖고 다른 엔진으로 아주 쉽게 갈아탈 수 있음
기타 장점내가 만든 거라서 내 맘대로 기능 추가가 가능. 자기도 나한테 말하면 아무 기능이나 추가 가능
기타 단점백엔드를 직접 구현해야함, 난 아직 못함...ㅠ
  1. 마크다운 문법을 이용해서 글을 작성한다.
    • 자기도 마크다운 문법 알테니까 이건 큰 문제 없음!
    • 근데 마크다운 확장 문법을 쓰기 때문에 기존 마크다운에 비해서 자유도도 높고 난이도도 높음
  2. 자체 제작한 엔진을 이용해서 내가 쓴 글을 html 파일로 변환한다.
    • 만약에 자기가 블로그 쓸 거면 내 엔진 쓰는 것보다는 더 유명한 엔진 쓰는게 나을 거 같아! 내 엔진은 딱히 사용자 친화적이지가 않아서..ㅎ
  3. 2번에서 만든 html 파일들을 통째로 github에 올린다. 그럼 github가 알아서 호스팅해줌
    • 내가 백엔드 구현할 줄을 몰라서 github 사용하는 중...
    • 이 방식의 장점
      • 사용이 쉽다. github에 모든 파일을 push 하기만 하면 됨
    • 이 방식의 단점
      • customizing이 어렵다.
      • 백엔드가 전혀 없기 때문에 할 수 있는게 없음 (ex: 검색기능 추가, 댓글 다는 기능 추가, 로그인 기능 추가 등등...)
